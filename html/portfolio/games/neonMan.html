Neon man (2024)
Brackeys Game Jam 2024.2 submission - #719/1480

Neon man is a short 2D survival-platformer game in which you have to explore, collect tokens and get back to a safe area before a storm rolls in. This game was made in one week for the 2024.2 Brackeys Game Jam, this was my second game jam and my first full game with Godot.

I solely developed this game over a week long period, all code, art, sfx and music was made by me.

❮ ❯

Concept

The theme for this jam was Calm before the storm, I had 2 main inspirations for the game :

  • Rainworld, from which the rain mechanic directly inspiring my storm mechanic and survival-exploration aspects of my game.
  • The Tron films and in-dev textures for games which led to me creating the minimalistic neon artstyle used in the game.

The main mechanic in Neon Man is the token and jump system, as you collect more tokens you upgrade your jump on the following day. These tokens are spread throughout the world which the player must find to progress through it.


Mechanics

Jump upgrades :

  • Jump upgrades is one of two core mechanics in the game, it allows for sectioning off different areas of the game by creating zones only accessible by upgrading the jump.
  • The player collects tokens spread throughout the level, when a certain amount is collected and the player sleeps through a storm, their jump will increase in height in the following day.
  • The amount of tokens needed for the jump changes for each jump allowing for control over how big an area can be, more tokens needed means a bigger area can be created to put the tokens in.

The storm :

  • A storm which rolls around every 2 minutes, which kills the player unless they are sheltering in a safe area.
  • Short timer due to small world size, if it were longer it may never occur in a normal playthrough as it would be too easy to get to next safe zone. Making it short promotes moving fast and taking risks making the storm feel more dangerous.
  • If development were to continue then a larger world size and timer could be implemented.
  • The effect was done with a shader in Godot's shading language which is similar to GLSL, it uses a mathematical function to create the effect.

Other elements :

  • There are many different elements in the world such as : spikes, platforms (stationary/moving/falling), fake walls/floors.
  • These all allow for more diverse and complex levels and secrets. This is important as it adds interesting challenges to the player and adds opportunities for rewarding exploration which is a key aspect of the game.

Difficulties and Reflection


This game was not very technically complex, the main difficulties from this game were the time constraints and new tools being used. Building a full game in a week means good organisation is needed, each aspect of the game needs time put toward it. This for the most part went well except that I neglected sound till the end resulting in little sound design and no music. The lack of music/sound was a major issue pointed out by players.

Another difficulty with a project with such a small time scale is the lack of proper testing leading to bugs and issues with gameplay, Neon man was (to my knowledge) bugless, even commented on by one player. There were issues with gameplay though the movement, it was very tight and unforgiving which wasn't the initial idea, this should have been planned better and implemented one way or the other. Potentially I could have adding some code to make it easier like in many platformers, having things like coyote time and jump buffers would make it feel more fair to the player. I did implement smaller hitboxes for the spikes with the aim to do this but more could have been done.
